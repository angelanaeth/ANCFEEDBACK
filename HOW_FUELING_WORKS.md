# 🔥 HOW FUELING WORKS - COMPLETE GUIDE

## 🎯 OVERVIEW

The fueling system automatically calculates **CHO (carbohydrates), hydration, sodium, and caffeine** for each planned workout in the next week (Monday-Sunday), then **writes the recommendations directly to TrainingPeaks pre-activity comments**.

---

## 📋 HOW IT WORKS - STEP BY STEP

### Step 1: Trigger Fueling Calculation

**API Endpoint**: `POST /api/fuel/next-week`

**Request**:
```bash
curl -X POST http://localhost:3000/api/fuel/next-week \
  -H "Content-Type: application/json" \
  -d '{"athlete_id":"427194"}'
```

**What Happens**:
1. Gets next Monday-Sunday date range
2. Fetches all PLANNED (not completed) workouts from TrainingPeaks
3. For each workout, calculates fueling needs
4. Queues workouts in database (`tp_write_queue` table)
5. Writes CHO notes back to TrainingPeaks pre-activity comments

---

### Step 2: Fueling Calculation Formula

**For each workout, the system calculates**:

#### A. **Carbohydrates (CHO)**
Formula based on workout duration and intensity:

```typescript
// Base rate: 60-120g/hour depending on intensity
const duration_hours = workout.Duration; // e.g., 2.5 hours
const intensity = workout.IntensityFactor || 0.75; // e.g., 0.75 = Z2

if (intensity < 0.55) {
  // Z1 - Low intensity
  carb_per_hour = 30;
} else if (intensity < 0.75) {
  // Z2 - Moderate
  carb_per_hour = 60;
} else if (intensity < 0.85) {
  // Z3-Z4 - Threshold
  carb_per_hour = 90;
} else {
  // Z5+ - VO2/Anaerobic
  carb_per_hour = 120;
}

total_carbs = carb_per_hour * duration_hours;
```

**Example**:
- Bike workout: 3 hours, Z2 (0.70 IF)
- CHO: 60g/hour × 3 hours = **180g total**
- Products: 6 gels (30g each) or 3 bottles of sports drink

#### B. **Hydration**
Formula based on duration, temperature, and sweat rate:

```typescript
// Base: 500-1000ml/hour
const temp_modifier = temperature > 25 ? 1.3 : 1.0;
const sweat_rate_modifier = athlete.sweat_rate || 1.0; // high/normal/low

fluid_per_hour = 750 * temp_modifier * sweat_rate_modifier;
total_fluid = fluid_per_hour * duration_hours;
```

**Example**:
- 3-hour workout, 20°C, normal sweat rate
- Fluid: 750ml/hour × 3 hours = **2,250ml total**
- Products: 3 bottles (750ml each)

#### C. **Sodium**
Formula based on sweat rate and duration:

```typescript
// Base: 200-500mg/hour
if (sweat_rate === 'high') {
  sodium_per_hour = 500;
} else if (sweat_rate === 'normal') {
  sodium_per_hour = 350;
} else {
  sodium_per_hour = 200;
}

total_sodium = sodium_per_hour * duration_hours;
```

**Example**:
- 3-hour workout, normal sweat rate
- Sodium: 350mg/hour × 3 hours = **1,050mg total**
- Products: 3 salt tablets (350mg each) or Precision Hydration

#### D. **Caffeine** (for key workouts)
```typescript
// 3-6mg per kg body weight
if (workout.isKeyWorkout || intensity > 0.90) {
  caffeine_mg = athlete.weight_kg * 4; // e.g., 70kg × 4 = 280mg
}
```

**Example**:
- Key workout, 70kg athlete
- Caffeine: **280mg** 30-45 minutes before start
- Products: 1 caffeine gel or 2 espressos

```

2. FRONTEND: Sends POST /api/fuel/next-week {"athlete_id": "427194"}

3. BACKEND:
   a. Gets next Monday-Sunday dates (e.g., Jan 13-19)
   b. Fetches planned workouts from TrainingPeaks API:
      GET /v2/workouts/427194/2026-01-13/2026-01-19
   
   c. For each planned workout:
      - Monday: Bike, 3hrs, 180 TSS
      - Wednesday: Run, 1.5hrs, 120 TSS
      - Friday: Swim, 1hr, 80 TSS
   
   d. Calculates fueling for each:
      Monday Bike:
        - CHO: 60g/hr × 3hrs = 180g
        - Fluid: 750ml/hr × 3hrs = 2,250ml
        - Sodium: 350mg/hr × 3hrs = 1,050mg
      
      Wednesday Run:
        - CHO: 60g/hr × 1.5hrs = 90g
        - Fluid: 750ml/hr × 1.5hrs = 1,125ml
        - Sodium: 350mg/hr × 1.5hrs = 525mg
      
      Friday Swim:
        - CHO: 30g/hr × 1hr = 30g (lower for swim)
        - Fluid: 500ml/hr × 1hr = 500ml
        - Sodium: 200mg/hr × 1hr = 200mg
   
   e. Writes each to tp_write_queue table
   
   f. Calls processFuelQueue() to write to TrainingPeaks:
      For each workout:
        PUT /v1/workouts/{workout_id}
        Body: { 
          PreActivityComments: "[fueling plan text]"
        }

4. TRAININGPEAKS: Updates workout with fueling notes

5. ATHLETE: Opens TrainingPeaks → Sees fueling plan in pre-activity comments
```

## 🔬 THE SCIENCE BEHIND IT

### Why These Numbers?

#### CHO (Carbohydrates)
- **30-60g/hour**: Z1-Z2 (easy endurance)
- **60-90g/hour**: Z3-Z4 (tempo/threshold)
- **90-120g/hour**: Z5+ (VO2/anaerobic)

**Source**: Sports nutrition research shows gut can absorb 60-90g/hour, trained athletes up to 120g/hour with multiple transport mechanisms (glucose + fructose).

#### Hydration
- **500-1000ml/hour**: Average sweat rate
- **Adjusted for**: Temperature, humidity, individual sweat rate

**Source**: ACSM guidelines recommend 400-800ml/hour, adjusted based on conditions.

#### Sodium
- **200-500mg/hour**: Replace sodium lost in sweat
- **Average**: 350mg/hour for most athletes

**Source**: Typical sweat contains 500-1200mg sodium per liter. Athletes lose 1-2L/hour, need to replace 30-50%.

#### Caffeine
- **3-6mg/kg**: Performance-enhancing dose
- **Timing**: 30-45 min pre-workout for peak effect

**Source**: Research shows 3-6mg/kg improves endurance performance by 2-4%.

```typescript
// In database, store athlete preferences:
{
  "athlete_id": "427194",
  "sweat_rate": "high",        // high/normal/low
  "carb_preference": "gel",     // gel/drink/real_food

  "sodium_needs": "high"        // high/normal/low
}

// Then fueling calculation adjusts:
if (athlete.sweat_rate === 'high') {
  fluid_per_hour *= 1.3;
  sodium_per_hour = 500;
}
```

### Adjust for Workout Type

```typescript
// Different sports have different needs:
if (workout.WorkoutType === 'Swim') {
  carb_per_hour *= 0.5;  // Lower for swim (no gut stress)
  fluid_per_hour *= 0.3;  // Pool environment
}

if (workout.WorkoutType === 'Run') {
  carb_per_hour *= 0.9;   // Harder to digest while running
  sodium_per_hour *= 1.2; // Higher sweat rate
}

if (workout.WorkoutType === 'Bike') {
  carb_per_hour *= 1.0;   // Standard (easiest to fuel)
}
```
