## 🎯 HOW IT WORKS

### Backend Flow

1. **Button Click** → POST `/api/fuel/next-week`
2. **Fetch Workouts** → GET TrainingPeaks planned workouts (next Mon-Sun)
3. **Calculate Fueling** → Brain.txt formulas
   - Carbs: 60-90g/hr (intensity-based)
   - Fluid: 300-600ml/hr (sport-based)
   - Sodium: 500-900mg/hr (duration-based)
4. **Queue to Database** → Insert to `tp_write_queue` table
5. **Background Worker** → `processFuelQueue()` runs async
6. **Write to TP** → PUT `/v2/workouts/plan/{id}` with Description field
7. **Mark Success** → Update queue status

### Frontend Flow

1. **User sees button** on athlete dashboard
2. **Clicks button** → Shows "Queueing..."
3. **Toast notification** → "✅ Fueling Queued: 8 workouts"
4. **Button shows ✓** → Success state for 3 seconds
5. **Background write** → Happens automatically
6. **Athletes see fueling** → In TrainingPeaks Pre-Activity Comments

## 📊 FUELING CALCULATION EXAMPLES

### Example 1: 90-Minute Zone 3 Bike Ride
```
Duration: 90 min (1.5 hrs)
Intensity: TSS/hr = 65 (Zone 3)
Sport: Bike
Weight: 70kg

Calculation:
- Carbs: 60-90g/hr range → 65% = 75g/hr → Total: 113g
- Fluid: 500ml/hr (bike) → Total: 750ml
- Sodium: 600mg/hr (90-120min) → Total: 900mg

Output:
🍌 CARBOHYDRATES: 113g (75g/hr)
💧 HYDRATION: 750ml (500ml/hr)
🧂 SODIUM: 900mg (600mg/hr)
```

### Example 2: 60-Minute Zone 5 Run
```
Duration: 60 min (1 hr)
Intensity: TSS/hr = 85 (Zone 5)
Sport: Run
Weight: 70kg

Calculation:
- Carbs: 90-120g/hr range → 100% = 90g/hr → Total: 90g
- Fluid: 600ml/hr (run) → Total: 600ml
- Sodium: 500mg/hr (<90min) → Total: 500mg

Output:
🍌 CARBOHYDRATES: 90g (90g/hr)
💧 HYDRATION: 600ml (600ml/hr)
🧂 SODIUM: 500mg (500mg/hr)
```
