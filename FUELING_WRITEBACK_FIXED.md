# ✅ FUELING WRITEBACK FIXED - UPDATES PREACTIVITYCOMMENTS CORRECTLY

**Date**: January 12, 2026  
**Issue**: Fueling data was overwriting workout descriptions instead of adding to Pre-Activity Comments

---

## 🐛 The Problem

### What You Saw:
- Fueling guidance being written to TrainingPeaks
- BUT it was **overwriting** the workout description
- Creating **companion workouts** with "[FUELING]" prefix
- **NOT adding** to Pre-Activity Comments field
- **Deleting** original workout info

### Example of Wrong Behavior:
```
❌ WRONG:
- Created new workout: "[FUELING] Sweet Spot Intervals"
- Overwrote Description with fueling data
- Original workout still there but cluttered
- Two workouts for same session
```

---

## ✅ The Fix

### What It Does NOW:
1. **Reads existing workout** from TrainingPeaks
2. **Gets current PreActivityComments** (preserves existing content)
3. **Appends fueling guidance** (doesn't overwrite)
4. **Updates via PUT** to `/v2/workouts/id/{workout_id}`
5. **Keeps original workout intact** (no companion workout)

### Example of Correct Behavior:
```
✅ CORRECT:
- Updates SAME workout (no new workout created)
- Adds to PreActivityComments field
- Preserves existing workout Description
- Preserves Title, TSS, Duration, etc.
- Appends fueling with separator
```

## 🔄 Update Flow

### Step 1: Read Workout
```javascript
GET /v2/workouts/{athlete_id}/{date}/{date}
- Gets all workouts for that date
- Finds workout by ID
- Retrieves current PreActivityComments
```

### Step 2: Build Updated Comments
```javascript
existingComments = workout.PreActivityComments || ''

if (existingComments.includes('ECHODEVO FUELING')) {
  // Replace old fueling with new
  finalComments = existingComments.replace(oldFueling, newFueling)
} else {
  // Append new fueling with separator
  finalComments = existingComments + separator + newFueling
}
```

### Step 3: Update Workout
```javascript
PUT /v2/workouts/id/{workout_id}
Body: {
  PreActivityComments: finalComments
}
```

### Step 4: Mark as Complete
```sql
UPDATE tp_write_queue 
SET status = 'success', 
    last_attempt = NOW()
WHERE id = {queue_id}
```
