# ⚡ FUEL NEXT WEEK FEATURE - COMPLETE

## 🎯 Overview

**Feature**: Automated Fueling Guidance for Planned Workouts
**Status**: ✅ COMPLETE (Backend + Frontend + Database)
**Purpose**: Automatically calculate and queue fueling guidance (carbohydrates, hydration, sodium) for next week's planned workouts

---

## 🚀 What It Does

1. **Fetches Next Week's Planned Workouts** (Monday → Sunday)
2. **Calculates Smart Fueling** using Brain.txt formulas:
   - Carbohydrates (g/hr) based on duration & intensity
   - Hydration (ml/hr) based on sport type & athlete weight
   - Sodium (mg/hr) for electrolyte replacement
3. **Queues for TrainingPeaks Write** to Pre-Activity Comments
4. **Shows Real-time Status** on dashboard

### Fueling Calculation Logic (from Brain.txt)

#### Carbohydrate Calculation
```
Duration < 60min:  0-30g/hr
Duration 60-90min: 30-60g/hr
Duration 90-120min: 60-90g/hr
Duration 120+min: 90-120g/hr

Adjusted for Intensity:
- TSS/hr > 80 (Zone 5+): Upper range
- TSS/hr > 60 (Zone 4): 75% of range
- TSS/hr > 40 (Zone 3): 50% of range
- TSS/hr < 40 (Zone 1-2): Lower range
```

#### Hydration Calculation
```
Run: 600ml/hr (higher sweat rate)
Bike: 500ml/hr
Swim: 300ml/hr (lower fluid needs)
```

#### Sodium Calculation
```
Duration > 120min: 800mg/hr
Duration > 90min: 600mg/hr
Duration < 90min: 500mg/hr
```

### Database Schema

#### Table: `tp_write_queue`
```sql
CREATE TABLE tp_write_queue (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  athlete_id TEXT NOT NULL,
  workout_id TEXT NOT NULL,
  workout_date TEXT,
  workout_title TEXT,
  workout_type TEXT,
  fuel_carb TEXT,
  fuel_fluid TEXT,
  fuel_sodium TEXT,
  status TEXT DEFAULT 'pending',
  attempts INTEGER DEFAULT 0,
  last_attempt TEXT,
  error_msg TEXT,
  created_at TEXT DEFAULT (datetime('now')),
  updated_at TEXT DEFAULT (datetime('now'))
);
```

#### Status Values
- `pending`: Queued, waiting to be processed
- `pending_api`: Logged, awaiting TrainingPeaks API endpoint
- `success`: Successfully written to TrainingPeaks
- `failed`: Failed to write (with error message)

### Button HTML

```html
<button onclick="fuelNextWeek('${athlete.id}'); return false;" 
        class="btn btn-danger btn-sm" 
        id="fuel-btn-${athlete.id}">
  <i class="fas fa-bolt me-1"></i>Fuel Next Week
</button>
```

### JavaScript Function
```javascript
async function fuelNextWeek(athleteId) {
  // 1. Show loading state
  btn.disabled = true;
  btn.innerHTML = '<i class="fas fa-spinner fa-spin me-1"></i>Queueing...';
  
  // 2. Call API
  const response = await axios.post('/api/fuel/next-week', { athlete_id: athleteId });
  
  // 3. Show success state
  btn.innerHTML = '<i class="fas fa-check me-1"></i>Queued!';
  
  // 4. Show toast notification
  showToast('success', `✅ Fueling Queued: ${response.data.queued} workouts`);
  
  // 5. Reset button after 3 seconds
  setTimeout(() => { /* reset */ }, 3000);
}
```

### Toast Notifications
- ✅ **Success**: Green toast with queued workout count
- ❌ **Error**: Red toast with error message
- **Auto-dismiss**: 5 seconds
